hexo with githiub action

通过公私钥实现私有仓库存储博客源代码，通过私有仓库 actions 操作将静态文件发布到公共仓库

只需要更新文本即可通过github action发布，也可以随时访问源码仓库

依赖安装

git

本机 git 已经安装，不赘述

npm

window 11 自带了choco，根据官网指引安装：https://nodejs.org/en/download/package-manager

ps：不熟悉window的环境变量设置，网上有一些方案是直接使用淘宝镜像安装，但搞不清楚怎么启用，感觉管理很混乱所以没有使用

创建仓库

  • 私有仓库：<name>-blog

    本地直接推送更新文件到该仓库，在该仓库配置 github actions，通过actions更新静态文件到公共仓库

  • 公共仓库：<name>.github.io

    存放 hexo 生成的静态页面

生成密钥

用来实现从私有仓库的actions中直接将更新后的静态文件push到公共仓库

建议将ssh密钥统一存放在 ~/.ssh 路径下便于管理，该命令会在当前路径生成公私钥文件，可以一路回车，也可自行配置：

  • 私钥：hexo-deploy-key

    • 配置到私有仓库：Settings -> Secrets and variables -> Actions -> New repository secret

  • 公钥： hexo-deploy-key.pub

    • 配置到公共仓库：Settings -> Deploy keys -> Add new

这里私钥的 key title 使用 HEXO_DEPLOY_SECRET，后续在 actions 配置中会用到

workflow 配置

私有仓库中配置 github actions workflow，创建文件 .github/workflows/deploy.yml

文件内容参考自：hexo workflow 参考，最简单的配置，删除了主题相关部分

hexo config 配置

在默认的 hexo _config.yml中，将 deploy 阶段的 target repo 设置为公有仓库地址

总结

整体流程不难，配置文件比较简单，但对nodejs、github actions，不熟悉还是花了点时间

私有仓库配置完成后，若 actions 失败参考日志输出 debug 即可

nodejs 版本尽量高，低版本会出现版本问题
